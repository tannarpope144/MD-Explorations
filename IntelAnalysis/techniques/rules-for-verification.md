# Rules for Verification

**Category:** Emerging analytic technique — structured
**Source:** ATP 2-33.4, *Intelligence Analysis* (Appendix A)

## Definition

Rules for Verification is a problem-solving methodology designed to establish **likelihoods, not certainties,** of hypotheses.

## When to use

Can be applied to:
- Situation development (updating intelligence assessments).
- Developing collection strategies.
- Conducting assessments.

## The seven rules

| Rule | Description |
|---|---|
| **Rule 1** | A hypothesis is more believable when a consequence of that hypothesis is verified. When a piece of supporting evidence is verified, the hypothesis becomes more credible. |
| **Rule 2** | The credibility of a hypothesis increases as different means used to test it support it — when various collection disciplines verify supporting evidence, or different sources within the same discipline corroborate. |
| **Rule 3** | Confidence increases as the observable bits of evidence that support it are in close proximity to each other. |
| **Rule 4** | Credibility is directly proportional to the number of instances in which it is supported — the more verifiable evidence, the greater the possibility of truth. |
| **Rule 5** | Confidence increases when an incompatible and rival conjecture is refuted. (Rarely is only one hypothesis considered.) |
| **Rule 6** | Confidence increases to the extent that it is consistent with another hypothesis that is highly credible. |
| **Rule 7** | Where observables support two different hypotheses, the simpler hypothesis stands a better chance of being true. |

> Rule 7 echoes [Occam's Razor](../reference/pitfalls.md#occams-razor) — prefer the less complex of two equally explanatory hypotheses.

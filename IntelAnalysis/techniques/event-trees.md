# Event Trees

**Category:** Basic structured analytic technique
**Source:** ATP 2-33.4, *Intelligence Analysis*

## Definition

An event tree is a graphic depiction of a possible sequence of events, including potential junctures within the event sequence.

## When to use

Use an event tree to clarify alternative event sequences with potential future (or at least unknown) outcomes related to an intelligence problem. Event trees work best when there are multiple, mutually exclusive options that cover the spectrum of reasonable alternatives.

An event tree is a visual tool by which analysts can depict a threat's options with decision points that provide insight into potential vulnerabilities. It clarifies the presumed sequence of events or decisions between an initiating event and a final outcome. Event trees also provide an excellent method for determining collection requirements for the indications that a decision has been made — or that events have unfolded in one of the alternative limbs of the tree.

## Method

1. Identify the mutually exclusive and complete set of possibilities that pertain to a given intelligence issue.
2. Decide which events, factors, or decisions (variables) will have the greatest influence on the alternatives or possibilities identified in Step 1.
3. Decide on the sequencing in which these factors are expected to occur or impact one another.
4. Determine the event options within each alternative and establish clear definitions for each event option to ensure that collection strategies to monitor events are effective.
5. Construct the event tree from left to right. Each alternative is a separate main branch. Start with the first alternative and create one branch from this node for each realistic path the first event can take. (Example: purchased equipment could be used for its intended purpose, reverse-engineered for duplication, or disassembled and sold for scrap.) Proceed down each event option node until the end state for that sub-branch is reached. Then move to the next alternative and repeat.
6. Determine what would indicate a decision has been made at each decision point for each option, to use in generating an integrated collection plan.
7. Assess the implications or aftereffects of each alternative on the intelligence problem.

## Tips

- Use in conjunction with [weighted ranking](weighted-ranking.md), hypothesis review techniques, and [subjective probability](subjective-probability.md) to gain added insights.
- Leverage the expertise of a group of analysts during construction to ensure all important events, factors, and decision options are considered.

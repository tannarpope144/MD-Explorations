# Matrices

**Category:** Basic structured analytic technique
**Source:** ATP 2-33.4, *Intelligence Analysis*

## Definition

A matrix is a grid with as many cells as required to sort data and gain insight. Matrices are useful whenever there are more options or more intricate data than can be conceptualized at one time without a visual representation. Whenever information can be reduced to a matrix, it provides analytic insights.

## When to use

Matrices are exceptionally useful for:

- Isolating critical data when there is an abundance of overall information relevant to an issue.
- Reviewing data related to options (such as the [analysis of competing hypotheses](analysis-of-competing-hypotheses.md)), enabling analytic focus on each option to improve comparison.
- Separating and categorizing the elements of a problem by type, facilitating comparison of different types of information or pieces of the same type.
- Identifying patterns or correlations within information — for example, telephone calls between members of a group (an intermediate step in [link analysis](link-analysis.md)).

A common application is cross-walking the operational variables memory aid **PMESII** (political, military, economic, social, information, infrastructure) with the civil considerations memory aid **ASCOPE** (area, structures, capabilities, organizations, people, events).

## Method

Matrices can be rectangular, square, or triangular, depending on purpose and the number of rows and columns required.

1. Draw a matrix with sufficient columns and rows to enter the two sets of data to be compared.
2. Enter the range of data or criteria along the horizontal and vertical axes.
3. In the grid squares between, note the relationships (or lack thereof) in the cell at the intersection between the two associated data points.
4. Review the hypotheses developed for the issue in light of the relationships shown in the matrix and, if appropriate, develop new hypotheses based on the insights gained.

## Tips

- Develop a template for recurring topics where the data points remain consistent.
- Color-code results to aid in understanding them.

## Cautions

- The two-dimensional design of matrices limits their use for collating data on complex issues.
- Omitting pertinent data can easily oversimplify an issue.
